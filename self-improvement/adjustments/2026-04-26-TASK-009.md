# TASK-009 Adjustment Log

## What changed
- Added an automatic calibration-review helper that counts settled paper bets, detects newly crossed 10-bet thresholds, runs `make chief -- calibration`, and writes a markdown review under `self-improvement/reviews/`.
- Hooked the helper into both paper-bet settlement paths so Chief settlement and Kalshi-driven daily settlement can trigger the report.
- Added unit coverage for threshold detection and review-file creation.

## Files changed
- `src/weatherlab/pipeline/calibration_reviews.py`
- `src/weatherlab/live/persistence.py`
- `src/weatherlab/settlement/kalshi_settlement.py`
- `tests/test_calibration_reviews.py`
- `self-improvement/adjustments/2026-04-26-TASK-009.md`

## Why
- Calibration review generation now happens automatically when settled paper bets cross 10, 20, 30, etc., removing the manual `make chief -- calibration` step at each milestone.

## Caveats
- The hook is non-blocking for settlements: if report generation fails, settlement still completes and the failure is logged.
- `make chief -- calibration` is executed from the repo root and uses the repo's Chief Makefile target/database configuration.
