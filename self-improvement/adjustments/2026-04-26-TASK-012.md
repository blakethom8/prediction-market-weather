# TASK-012 Adjustment Log

## What changed
- Added a macro Kalshi scan script that fetches all open markets, filters for low-priced liquid candidates, formats a scannable report, and optionally sends it through `openclaw system event`.
- Added a general `KalshiClient.fetch_open_markets()` method so macro scanning does not reuse the weather-only fetch path.
- Normalized Kalshi cent-style prices into probability prices in live market ingestion.
- Added unit coverage for candidate filtering, report formatting, empty reports, and all-open-market fetch normalization.

## Files
- `scripts/macro_scan.py`
- `src/weatherlab/ingest/kalshi_live.py`
- `tests/test_macro_scan.py`
- `tests/test_kalshi_live.py`

## Why
Macro structural markets can produce ColdMath-style asymmetric setups without depending on weather forecast accuracy. The scanner surfaces `yes_ask <= 0.15` and `volume > 500` markets for human review without making recommendations.

## Caveats
- The script scans all open Kalshi markets, so live runtime depends on Kalshi pagination size and API latency.
- The report is informational only; it does not score, recommend, or place trades.
