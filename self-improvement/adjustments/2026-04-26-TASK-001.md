# TASK-001: Apply warm bias correction to forecast pipeline

## Changed

- Added `WARM_BIAS_CORRECTION_F = 1.5` in `src/weatherlab/forecast/asos.py`.
- Applied the correction to the ASOS morning validation `forecast_high_f` immediately after the NWS forecast is fetched, before confidence scoring and before downstream probability selection can consume the value.
- Kept the corrected value as a decimal Fahrenheit value so `+1.5°F` is preserved for threshold-sensitive probabilities.
- Added ASOS unit coverage verifying that a raw 52°F forecast becomes 53.5°F.

## Files

- `src/weatherlab/forecast/asos.py`
- `tests/test_asos.py`
- `self-improvement/adjustments/2026-04-26-TASK-001.md`

## Why

The post-mortem in `self-improvement/insights/2026-04-23-systematic-cool-bias.md` found a systematic cool bias across settled edge bets, likely because airport settlement stations run warmer than generic metro-area NWS forecast points. The immediate mitigation is a conservative global `+1.5°F` additive correction.

## Caveats

- This is a global correction. The related insight notes the bias likely varies by city, especially LAX.
- The correct longer-term fix is to calibrate or configure station-specific forecast inputs/corrections.
