# TASK-005 Adjustment Log

## What Changed
- Raised the ColdMath proposal scan gap floor from 8F to 10F.
- Updated the morning scan to use the same 10F floor when generating `coldmath_plays`.
- Updated the no-play scan report text so it reports the active 10F threshold.

## Files

## Why
Small post-correction forecast gaps are no longer worth proposing for real-money ColdMath. The platform should only generate ColdMath proposals when the already bias-corrected `forecast_high_f` leaves at least a 10F gap to the contract threshold or bucket edge.

## Caveats
- `fetch_morning_validation` already applies the warm-bias correction before `forecast_high_f` reaches the ColdMath scan, so this change keeps the threshold check downstream of that correction.
- The auto-bet ColdMath validator was already configured at 10F; this change aligns proposal generation with that existing real-money gate.
